发表于 2019-04-03 | 分类于 Hexo

问题描述

我在vscode里用markdown编辑博客时，数学公式可以在编辑的preview中可以展现出来，但是发布到GitHub上，用游览器访问博客页面时，则是公式的LaTeX码，如下：

问题原因

参考链接1

Hexo默认使用”hexo-renderer-marked”引擎渲染网页，该引擎会把一些特殊的Markdown符号转换为相应的html标签，比如在Markdown语法中，下划线代表斜体，会被渲染引擎处理为特殊标签。
因为类Latex格式书写的数学公式下划线表示下标，有特殊的含义，如果被强制转换为特殊标签，那么MathJax引擎在渲染数学公式的时候就会出错。

解决方案

1.更换Hexo的markdown渲染引擎，hexo-renderer-kramed引擎是在默认的渲染引擎hexo-renderer-marked的基础上修改了一些bug，两者比较接近，也比较轻量级。
